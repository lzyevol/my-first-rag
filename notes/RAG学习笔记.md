# RAG 学习笔记

## RAG 是什么
RAG（Retrieval-Augmented Generation，检索增强生成）是一种让大语言模型在回答问题前先去外部知识库查资料的技术。它不像微调那样修改模型参数，而是给模型配一个"外挂知识库"。

核心思想：将大模型内部的参数化知识与外部的非参数化知识相结合。

## RAG 的四步流程
构建一个最小可用的 RAG 系统需要四个步骤：数据准备、索引构建、检索优化、生成集成。

1. 数据准备：加载文档（PDF/Word/Markdown），切成小段
2. 索引构建：用嵌入模型把每段文字转成向量，存入向量数据库
3. 检索：把用户问题也转成向量，在库里找最相似的段落
4. 生成：把搜到的段落和用户问题一起发给大模型，生成答案

## 文本分块策略
文本分块是 RAG 的关键步骤，主要有三种策略：

1. 固定大小分块：简单按字符数切，可能破坏语义
2. 递归字符分块：按分隔符优先级逐级切（段落 → 句子 → 字符），最常用
3. 语义分块：用嵌入模型判断语义断点，在话题变化处切分

分块时要注意 chunk_overlap（块间重叠），避免关键信息在边界处断裂。

## 向量嵌入
Embedding 是把文字转成一串数字（向量）的技术。语义相近的文字，向量坐标也相近。最常用的相似度度 量方式是余弦相似度。主流中文嵌入模型有 BGE 系列（BAAI/bge-small-zh-v1.5 等）。

## 向量数据库
向量数据库专门存储和搜索向量。常用的有：
- FAISS：Meta 出品，本地文件存储，适合小项目和学习
- Chroma：轻量级，零配置
- Milvus：分布式，适合生产环境大规模部署

## 混合检索
混合检索结合了两种搜索方式：
1. 稀疏向量（BM25）：关键词精确匹配，擅长专有名词
2. 密集向量（语义向量）：理解同义词和上下文，擅长模糊语义搜索

两种结果通过 RRF（倒数排序融合）合并排序，取长补短。

## RAG 与大模型微调的区别
- RAG：不修改模型参数，给模型外挂知识库，解决"模型不知道什么"的问题
- 微调：修改模型参数，改变模型的行为风格/输出格式，解决"模型怎么做"的问题

技术选型顺序：提示工程 → RAG → 微调，优先选择对模型改动最小的方案。

## RAG 系统评估
评估 RAG 系统有三个核心维度（RAG 三元组）：
1. 上下文相关性：检索出来的内容和问题相关吗
2. 忠实度：生成的答案有没有在上下文中找到依据（防幻觉）
3. 答案相关性：最终答案完整、直接地回答了用户问题吗

## Function Calling
Function Calling 是让大模型能调用外部工具的能力。模型不是直接回答，而是输出一个结构化的调用指令（函数名+参数），由程序执行后把结果返回给模型，模型再生成最终答案。这是构建 Agent 的核心基础。
